# 技能：计划索引管理（SKILL_PLAN_INDEX）

> 用途：规范计划的完整生命周期——创建、更新、完成、撤销，确保计划索引始终准确、单一、可追溯。
> 适用场景：任何涉及计划状态变化的操作。

---

## 权威源声明

> **计划索引唯一维护于 `AGENTS.md`**。
> 不在 `CLAUDE.md`、`developers/INDEX.md` 或其他文件中重复维护计划表格。
> 其他文件若需引用，只写"见 `AGENTS.md` 计划索引"。

---

## 计划索引格式

位置：`AGENTS.md` 末尾"计划索引"区块。

```markdown
| 计划ID | 计划文档 | 最后更新时间 | 完成情况 | 验收状态 |
|---|---|---|---|---|
| PLAN-XXX-001 | `路径/文件.md` | YYYY-MM-DD | 规划中 | 未验收 |
```

### 字段说明

| 字段 | 规则 |
|---|---|
| 计划ID | 格式 `PLAN-[主题缩写]-[序号]`，全局唯一，一旦创建不可修改 |
| 计划文档 | 指向实际存在的计划文件路径 |
| 最后更新时间 | 每次状态变化时更新为当天日期 `YYYY-MM-DD` |
| 完成情况 | 见下方状态流转 |
| 验收状态 | `未验收` / `已验收` / `部分验收（说明）` |

### 完成情况状态流转

```
规划中 → Phase N 进行中 → Phase N 完成 → 全部完成 → （撤销）
```

## 操作步骤

### 创建计划

1. 新建计划文档（如 `FEATURE-PLAN.md`），写明目标、分阶段任务、DoD。
2. 在 `AGENTS.md` 计划索引表中新增一行：
   - 计划ID：`PLAN-[主题]-[下一个序号]`
   - 完成情况：`规划中`
   - 验收状态：`未验收`
3. 追加会话留痕。

### 更新计划进度

1. 在 `AGENTS.md` 计划索引找到对应行。
2. 更新"完成情况"和"最后更新时间"。
3. 追加会话留痕，注明当次变更内容。

### 完成计划（全部）

1. 执行 [`SKILL_ACCEPTANCE.md`](./SKILL_ACCEPTANCE.md) 收束验收。
2. 验收通过后，在 `AGENTS.md` 计划索引更新：
   - 完成情况：`全部完成`
   - 验收状态：`已验收`
3. 追加会话留痕，包含验收结论摘要。

### 撤销计划

1. 在 `AGENTS.md` 计划索引更新：
   - 完成情况：`已撤销（原因：…）`
   - 验收状态：`不适用`
2. 计划文档**保留不删除**，顶部增加撤销说明。
3. 追加会话留痕。

---

## 完成后必做

按 [`SKILL_ROUTER.md`](./SKILL_ROUTER.md) 的"计划创建/进度更新/完成"行：

1. **更新 `AGENTS.md` 计划索引**（以上操作步骤即为此项）。
2. **会话留痕**：在 `developers/SESSIONS/NOTE_YY_MM_DD.md` 追加：

```
### [计划管理] [操作类型：创建/更新/完成/撤销] [计划ID]
- 时间：YYYY-MM-DD HH:mm
- 操作详情：…
- 当前状态：完成情况 / 验收状态
- 遗留事项：…
```

## doctor 检测项

`project-rules-kit doctor` 会对以下情况告警：

- `CLAUDE.md` 或 `developers/INDEX.md` 中含独立计划索引表（口径冲突）
- `AGENTS.md` 中计划文档路径指向不存在的文件（断链）

## 常见问题

**Q：计划文档放在哪里？**  
A：无固定目录要求，推荐放在项目根目录或 `developers/PLANS/`（如有）。重要的是 `AGENTS.md` 中的路径指向正确。

**Q：同一天有多个计划状态变更怎么记？**  
A：追加到同一天的 SESSIONS 文件，每个变更单独一个 `###` 条目。

**Q：已有项目计划索引分散在多处怎么迁移？**  
A：执行 `doctor` 识别冲突位置 → 将内容合并到 `AGENTS.md` → 删除其他文件中的计划表格 → 补充留痕。
