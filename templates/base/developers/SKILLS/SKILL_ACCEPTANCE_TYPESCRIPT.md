# 技能：计划目标验收 — TypeScript 通用项目

> 适用栈：TypeScript CLI、SDK、Node.js 工具、前后端共享库、纯模块库等非特定前端/后端形态项目。
> 上级入口：[SKILL_ACCEPTANCE.md](./SKILL_ACCEPTANCE.md)

---

## 一、功能目标核对

对照计划文档，逐条确认以下问题：

- [ ] 计划中声明的导出能力（命令、函数、类、接口、工具模块）均已实现
- [ ] 对外导出名称、参数、返回值与计划或设计约定一致
- [ ] 外部输入（CLI 参数、配置、环境变量、文件内容、网络数据）已完成显式校验
- [ ] 核心业务分支（成功路径 + 失败路径）均已实现，不依赖调用方兜底
- [ ] 计划中声明的副作用行为（文件写入、网络调用、进程退出码、日志输出）符合预期
- [ ] 如涉及公共类型，类型定义已覆盖主要使用场景，未暴露明显不安全的 `any`

---

## 二、代码质量检查

```bash
# 静态检查
npm run lint

# 类型检查
npx tsc --noEmit

# 单元测试 / 集成测试
npm run test

# 构建验证（如项目提供）
npm run build
```

**判定要求：**
- lint 无 error
- TypeScript 无类型错误
- 与计划目标直接相关的测试全部通过
- 若项目包含构建产物，构建过程无报错且产物可正常生成

## 三、非功能检查

### 3.1 类型安全

- [ ] 公共 API 的参数与返回值类型明确，不依赖隐式 `any`
- [ ] 联合类型、可选字段、泛型边界已覆盖真实分支，不存在明显类型逃逸
- [ ] 类型断言（`as`）仅用于边界收敛，不把不确定输入直接断言为目标类型

### 3.2 健壮性与错误处理

- [ ] 错误信息包含可定位上下文（参数名、文件路径、模块名、阶段）
- [ ] 不存在静默吞错；捕获异常后有重抛、包装或统一上报
- [ ] 资源清理完整（临时文件、定时器、连接、流、订阅）

### 3.3 可维护性

- [ ] 模块职责清晰，无明显“巨型文件”继续扩张
- [ ] 重复逻辑已抽取为共享函数/模块，而非复制粘贴
- [ ] 对外能力有最小必要文档或示例，调用方式可被团队复现

### 3.4 工程基线

- [ ] `package.json` 中脚本与实际检查命令一致，无失效脚本
- [ ] 构建输出、声明文件、入口字段（`main` / `module` / `exports` / `types`）与计划目标一致
- [ ] 环境变量或配置项存在文档或示例，不依赖隐式本地环境
