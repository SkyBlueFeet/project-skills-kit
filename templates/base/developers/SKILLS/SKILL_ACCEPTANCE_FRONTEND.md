# 技能：计划目标验收 — 前端

> 适用栈：React / Vue / Svelte / 原生 HTML+CSS+JS / 小程序等 Web 前端项目。
> 上级入口：[SKILL_ACCEPTANCE.md](./SKILL_ACCEPTANCE.md)

---

## 一、功能目标核对

对照计划文档，逐条确认以下问题：

- [ ] 计划中声明的所有页面/视图均已实现，路由可访问
- [ ] 各页面的交互功能（表单提交、列表操作、弹窗、跳转）与设计一致
- [ ] 数据获取逻辑（接口调用、参数、字段映射）符合计划约定
- [ ] 加载态、空态、错误态（接口失败）均有处理，不出现白屏
- [ ] 计划中涉及的权限控制（路由守卫、元素级隐显）已落地
- [ ] 国际化、主题切换等计划内全局特性已实现（如有）

---

## 二、代码质量检查

```bash
# 格式化与静态检查
npm run lint
npm run format:check  # 如有

# 类型检查（TypeScript 项目）
npx tsc --noEmit

# 单元测试
npm run test          # Jest / Vitest

# 构建
npm run build
```

**判定要求：**
- lint 无 error
- TypeScript 无类型错误
- 与计划目标直接相关的组件测试通过
- 生产构建无报错，产物体积无异常膨胀

## 三、非功能检查

### 3.1 用户体验基线

- [ ] 所有用户操作有响应反馈（loading / toast / 禁用态），无"点了没反应"情况
- [ ] 表单有前端校验，错误提示明确，定位到具体字段
- [ ] 键盘导航与焦点管理不破坏（Tab 顺序合理，弹窗焦点锁定）
- [ ] 移动端/响应式断点下布局无明显溢出或遮挡（如计划有此要求）

### 3.2 可访问性（Accessibility）

- [ ] 图片有 `alt` 属性
- [ ] 交互元素可通过键盘触发（`button`、`a` 等语义元素，或配 `role` + 键盘事件）
- [ ] 颜色对比度不低于 AA 标准（仅在计划有无障碍要求时强制）

### 3.3 性能基线

- [ ] 首屏关键资源无明显大文件（图片已压缩，无未拆包的巨型 JS）
- [ ] 路由懒加载已配置（SPA 项目）
- [ ] 无明显内存泄漏风险（事件监听、定时器在组件卸载时已清理）

### 3.4 安全基线

- [ ] 用户可控内容渲染不使用 `dangerouslySetInnerHTML` / `v-html`，或已做转义
- [ ] 敏感信息（token、密钥）不写入 `localStorage` 明文（如有安全要求）
- [ ] 外部跳转链接加 `rel="noopener noreferrer"`

### 3.5 工程规范

- [ ] 组件职责单一，无超过 300 行的"巨型组件"（参考项目规范阈值）
- [ ] 公共逻辑已抽取为 Hook / Composable / 工具函数，无跨组件复制粘贴
- [ ] 环境变量通过 `.env` 文件管理，未在源码中硬编码接口地址

## 四、视觉走查（人工）

以下项目需人工确认（Agent 辅助提示，不可自动判定）：

- [ ] 与 UI 设计稿的关键页面视觉对齐（间距、字色、图标）
- [ ] 动画/过渡效果流畅，无跳帧
- [ ] 浏览器兼容性（目标浏览器范围内无明显问题）
