# 技能：计划目标验收 — JavaScript 通用项目

> 适用栈：JavaScript CLI、SDK、Node.js 工具、脚本、纯模块库等非特定前端/后端形态项目。
> 上级入口：[SKILL_ACCEPTANCE.md](./SKILL_ACCEPTANCE.md)

---

## 一、功能目标核对

对照计划文档，逐条确认以下问题：

- [ ] 计划中声明的导出能力（命令、函数、类、工具模块）均已实现
- [ ] 对外导出名称、参数约定、返回结构与计划或设计一致
- [ ] 外部输入（CLI 参数、ENV、JSON、文件内容、网络返回）已完成显式校验
- [ ] 核心业务分支（成功路径 + 失败路径）均已实现，不依赖调用方兜底
- [ ] 计划中声明的副作用行为（文件写入、网络调用、进程退出码、日志输出）符合预期
- [ ] 模块边界清晰，不在单个文件混合参数解析、业务编排与 I/O 实现

---

## 二、代码质量检查

```bash
# 静态检查
npm run lint

# 单元测试 / 集成测试
npm run test

# 构建验证（如项目提供）
npm run build
```

**判定要求：**
- lint 无 error
- 与计划目标直接相关的测试全部通过
- 若项目包含构建/打包步骤，构建过程无报错且产物可正常生成

## 三、非功能检查

### 3.1 健壮性与错误处理

- [ ] 外部输入校验完整，非法输入返回或抛出明确错误
- [ ] 错误信息包含可定位上下文（模块、参数、阶段）
- [ ] 不存在静默吞错；捕获异常后有重抛、包装或统一上报

### 3.2 可维护性

- [ ] 模块职责清晰，无明显“巨型文件”继续扩张
- [ ] 重复逻辑已抽取为共享函数/模块，而非复制粘贴
- [ ] 公共能力有最小必要文档或示例，调用方式可被团队复现

### 3.3 工程基线

- [ ] `package.json` 中脚本与实际检查命令一致，无失效脚本
- [ ] 入口字段（`main` / `module` / `exports` / `bin`）与计划目标一致
- [ ] 环境变量或配置项存在文档或示例，不依赖隐式本地环境

### 3.4 运行稳定性

- [ ] 临时文件、定时器、连接、流、订阅等资源已正确清理
- [ ] 异步流程不存在明显未处理的 Promise 拒绝
- [ ] 如为 CLI，退出码与错误输出符合预期，成功/失败场景可区分
