# 分析文档目录

本目录存放对代码、需求、技术方案的分析记录。

## 命名规则

- `ANALYSIS_[主题].md`（主题使用大写与短横线分隔）

示例：
- `ANALYSIS_ARCHITECTURE-BOUNDARY.md`
- `ANALYSIS_BUILD-PIPELINE.md`
- `ANALYSIS_ERROR-HANDLING.md`

## 文档格式

参见 [DOC-RULES.md](../DOC-RULES.md)。

每份分析文档必须包含：
- 版本锚点（创建时间、最后更新、代码快照日期、Git 信息）
- 参考代码表（列出分析时阅读的文件）
- 结论/建议（明确是否需要后续改动）
