# Python 代码规范（工程执行版）

## 1. 组织与职责

- 按领域拆分模块，禁止单文件承载多类职责。
- 核心逻辑与 I/O、框架胶水代码分离。

## 2. 风格

- 遵循 PEP 8 与类型标注（公共函数必须标注）。
- 函数命名用动词短语，常量全大写蛇形命名。
- 优先小函数与早返回，避免深层嵌套。

## 3. 健壮性

- 外部输入必须校验（类型、空值、范围、格式）。
- 禁止裸 `except:`；异常捕获必须指定类型并保留上下文。
- I/O 资源用上下文管理器管理。

## 4. 测试与门禁

- 每个新增能力至少包含 1 条成功和 1 条失败路径测试。
- 建议命令：`ruff check .`、`pytest`。
