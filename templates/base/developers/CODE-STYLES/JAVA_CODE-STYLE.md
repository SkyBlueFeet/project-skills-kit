# Java 代码规范（工程执行版）

## 1. 结构与分层

- 包结构按领域划分：`api`、`service`、`infra`、`model`。
- 避免“巨型 Service”；一个类只承担一种稳定职责。

## 2. 命名与可读性

- 类名使用名词，方法名使用动词。
- 公共 API 必写 Javadoc：用途、参数、返回值、异常语义。
- 优先组合，慎用继承。

## 3. 健壮性

- 对外输入必须校验（`null`、范围、格式）。
- 异常分层：业务异常与系统异常分离。
- 资源必须显式释放（流、连接、句柄）。

## 4. 测试与门禁

- 新增能力至少含成功与失败路径测试各 1 条。
- 建议命令：`mvn -q test` 或 `gradle test`。
