# JavaScript 代码规范（工程执行版）

## 1. 模块与职责

- 按能力分层：`core`、`runtime`、`adapters`、`tools`。
- 禁止在一个文件混合参数解析、业务编排、I/O 实现。

## 2. 代码风格

- 默认使用 ES Modules。
- 函数命名使用动词短语，布尔值使用 `is/has/can/should` 前缀。
- 单函数建议不超过 40 行；超过 60 行必须拆分。

## 3. 健壮性

- 外部输入必须显式校验（CLI、ENV、JSON、网络返回）。
- 异常必须携带上下文字段（模块、参数、阶段）。
- 禁止静默吞错；捕获后必须重抛或统一上报。

## 4. 测试与门禁

- 新增能力至少包含 1 条成功路径 + 1 条失败路径测试。
- 建议命令：`npm run lint`、`npm test`。
