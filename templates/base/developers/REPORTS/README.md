# 质量报告目录说明

本目录用于存放自动生成或人工整理的代码质量检查报告。

## 1. 命名规则

- 文件名：`CODE_QUALITY_REPORT_YYYY-MM-DD_HH-mm-ss.md`

## 2. 必填章节

1. 报告元信息
2. 检查标准
3. 自动化检查结果
4. 结论与建议
5. 命令输出摘录

## 3. 必填字段

- 检查时间（`YYYY-MM-DD HH:mm:ss +08:00`）
- 检查范围（按项目实际目录）
- 标准来源（当前任务要求 + `developers/CODE-STYLE.md`）
- 各标准通过/失败状态
- 失败项明细（命令与关键错误）
- 总体判定（`PASS` / `FAIL`）

## 4. 判定标准（通用）

- 已执行该项目语言栈对应的格式化、静态检查、测试、构建。
- 对关键功能路径给出可复现验证结果。
